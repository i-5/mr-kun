MRKun Taglibrary Reference

This is a reference guide for the utility JSP tags that you should use for the MRKun Project.
Note that, with the exception of one tag, this is a general taglibrary that can be used on any project.

The Tag Library <taglib>

The <taglib> contains generic tags you can use to create JSP pages.

To use tags from this library in your JSP page you must add the following line at the top of the JSP page:
<%@ taglib uri="/taglib.tld" prefix="tt" %>

<tt:condition>

The <tt:condition> tag is used to evaluate an expression. The body of the tag is executed if the expression evaluates to value.

 
Attribute Required? Default
value		 Description
expression

yes

  

The expression to evaluate.
value

yes

  

The value to compare the result of the expression to.
ignoreCase

no

 false

If true, ignore case when comparing the result of the expression to value.
negate

no

 false

If true, negates the evaluation of the tag.

Example:

<tt:condition expression="<%= myName %>" value="nicklas" ignoreCase="true">
  Hi Nicklas!
</tt:condition>

<tt:date>

The <tt:date> tag is used to display the current date in a specified format.

 
Attribute Required? Default
value		 Description
pattern

yes

  

The pattern used to format the date. The syntax of this pattern is the same as used in the Java class SimpleDateFormat.

Example:

<tt:date pattern="MM/dd/yy hh:mm:ss a" />

<tt:dateFormat>

The <tt:dateFormat> tag is used to display a date in a specified format.

 
Attribute Required? Default
value		 Description
date

yes

  

The date to format. This object must be a Date class, not a string.
pattern

yes

  

The pattern used to format the date. The syntax of this pattern is the same as used in the Java class SimpleDateFormat.
locale

no

 default Locale

The locale used when formatting the date.

Example:

<tt:dateFormat date="<%= myDate %>" pattern="yyyy-MM-dd" />

<tt:decimalFormat>

The <tt:decimalFormat> tag is used to display a number in a specified format.

 
Attribute Required? Default
value		 Description
number

yes

  

The number to format. This object must be a subclass of Number, not a string.
pattern

yes

  

The pattern used to format the number. The syntax of this pattern is the same as used in the Java class DecimalFormat.
locale

no

 default Locale

The locale used when formatting the number.

Example:

<tt:decimalFormat number="<%= myNumber %>" pattern="#,##0.00" />

<tt:isEmpty>

The <tt:isEmpty> tag is used to check to see if a collection is empty. The body of the tag is executed if the collection is empty.
 
Attribute Required? Default
value		 Description
collection

yes

  

The name of the collection.

Example:

<tt:isEmpty collection="my_list">
  my_list is empty!
</tt:isEmpty>

<tt:isNull>

The <tt:isNull> tag is used to evaluate an expression. The body of the tag is executed if the expression is null.
Or in the case of a string, if the string is null or empty.

Since you can't do an if-else construct in JSP, you often have to use isNull together with the notNull tag.
 
Attribute Required? Default
value		 Description
expression

yes

  

The expression to evaluate.

Example:

<tt:isNull expression="<%= myObject %>">
  myObject is null!
</tt:isNull>

<tt:iterate>

The <tt:iterate> tag is used to iterate over a collection.

 
Attribute Required? Default
value		 Description
collection

yes

  

The collection to iterate over. This object must be a subclass of Collection.
type

yes

  

The complete class name of the elements in the collection.
variable

yes

  

The name of a variable that holds the current element in the collection. The variable can be referenced inside the body of the tag.
counterVariable

no

  

The name of a variable that holds the number of the current element in the collection. The variable can be referenced inside the body of the tag.
start

no

 0

The start value for the counterVariable. This only affects the counterVariable. The iterate tag always goes through the entire collection.

Example:

<tt:iterate collection="offering_list" type="com.eoffering.server.PublicOfferingData" variable="publicOffering">
  <tr>
  <td><%= publicOffering.getOfferingType() %></td>
  </tr>
</tt:iterate>

<tt:link>

The <tt:link> tag is used to display an HTML link.

The url is passed through encodeURL to enable users without cookies. Therefore you should always use this tag when you have links in your JSP page.

If the body of the tag is empty, the url link is used as the text of the link.

 
Attribute Required? Default
value		 Description
url

yes

  

The url of the link.

Examples:

<tt:link url="www.acme.com">ACME</tt:link>
<tt:link url="www.eoffering.com" />

<tt:loop>

The <tt:loop> tag is used to create a loop.

 
Attribute Required? Default
value		 Description
start

no

 0

The start value for the loop.
end

yes

  

The end value for the loop.
counterVariable

no

  

The name of a variable that you can reference inside the body of the tag.

Example:

<tt:loop start="1" end="3" counterVariable="i">Looping: <%= i %></tt:loop>

<tt:messageFormat>

The <tt:messageFormat> tag is used to display a message with specified parameters.

 
Attribute Required? Default
value		 Description
arguments

yes

  

An array of arguments for the message.
pattern

yes

  

The pattern used to format the message. The syntax of this pattern is the same as used in the Java class MessageFormat.

Example:

<tt:messageFormat pattern="Error {0} occurred. The message is: {1}" arguments="<%= errorArgs %>" />
